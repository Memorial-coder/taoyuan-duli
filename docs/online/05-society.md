# 村社线

- 主目标：把玩家关系从互助推进到组织、提案、投票与公共建设。
- 首批范围：村社创建、成员角色、公告、提案、投票、公共建设记录。
- 首批目标文件：`taoyuan-main/src/views/game/SocietyView.vue`、`taoyuan-main/src/stores/useSocietyStore.ts`、`server/src/taoyuanSocietyRuntime.js`。
- 边界：村社共治只驱动公共层与声望层，不直接改写他人的单人经营明细。

## 当前进度

- `L70` 第一轮已完成：村社当前已经有独立在线入口，支持创建村社、保存名称、简介、徽记、主题、公开范围、成员容量与入社条件，不再和单机公会系统混用同一条实现线。
- 村社创建者现在会自动成为村社社长，村社卡片也会保留最近动态、基础成员列表和创建时间，为后续职位治理、提案与投票继续扩展留出稳定结构。
- `/game/society` 页面、主导航和移动地图菜单都已接通第一版“村社”入口；现在可以查看自己的村社，并浏览其他公开 / 半公开村社的基础卡片。
- 在线 smoke 已补进 `L70` 回归：当前会实际验证村社总览读取、创建村社、创建者回读自己的村社，以及其他账号从公开列表回看到这张新建村社卡片。
- `L71-L72` 第一轮已完成：村社当前已经补齐六种职位 `president / steward / buyer / treasurer / scribe / member`，并打通申请加入、邀请加入、接受 / 拒绝请求、职位调整、公告更新、提案创建、投票、归档和历史回读。
- 权限口径也已经固定：`president + steward` 负责邀请与审核入社，`president + steward + scribe` 负责公告维护与提案归档，只有 `president` 可以调整成员职位，继续优先保护单人经营主循环，不把他人个人经营细项直接暴露给村社共治层。
- `SocietyView`、`useSocietyStore` 与在线接口已经完成第一轮治理承接：现在可以在同一页查看自己村社、处理申请 / 邀请、编辑公告、调整职位、发起提案、投票、归档和查看历史提案，公开村社列表也已支持直接申请加入。
- 在线 smoke 已补进 `L71-L72` 回归：当前会实际验证村社读写、申请加入、接受请求、职位轮转、公告写回、提案创建、投票、归档与历史提案回读，第一轮村社治理链已经具备可重复回归能力。
- `L73` 第一轮已完成：村社当前已经补齐八项公共建设工程 `修桥 / 修码头 / 建集市 / 建书院 / 修灯街 / 扩仓库 / 修温泉 / 建祠堂`，并把工程进度、捐献记录、个人贡献、完工反馈和世界变化反馈接进同一套在线村社 runtime。
- 公共建设目前采用固定捐献包口径：成员可以直接在村社页提交工钱、木料、石料或图纸文书等捐献包，服务端会真实扣减当前账号的在线存档铜钱与材料，再把推进进度、个人贡献和最近捐献回写到村社工程面板。
- 第一轮公共建设仍然坚持边界收口：它只改变在线村社公共层的工程状态和反馈，不直接改写单机 `VillageProject` 主链，也不把他人的个人经营明细暴露给村社层；这样可以先把共建闭环收稳，再考虑更深层的世界联动。
- 在线 smoke 已补进 `L73` 回归：当前会实际验证公共建设捐献写路、服务端存档扣减、工程进度推进与公共建设回读，村社共建链已经第一次进入可重复回归。
- `L74` 第一轮已完成：当前继续复用现有职位、提案与公共建设模型，把村社等级、福利、公共仓和专属权益都收在同一套在线村社数据结构里，而不是重新拆第二套组织成长系统。
- 村社等级当前按“已完成公共建设 + 公共建设贡献次数 + 公共仓入仓次数”自动换算，不额外引入手工升级按钮；村社快照会同步返回等级称号、当前福利经验、累计福利经验和距下一等级差值。
- 村社福利当前先做成“解锁清单 + 权益预热”口径：公告常驻、公共仓、专属节会优先、专属装饰位、专属任务板和村社史册都会按等级显示解锁状态，先把成长反馈接进同一页。
- 公共仓已经接成真实写路：成员可以在村社页直接执行木料 / 石料 / 纸张 / 草药 / 公用经费入仓，服务端会真实扣减当前账号在线存档里的铜钱与材料，再把公共仓资金、物资和入仓日志写回村社层。
- 专属节会、专属装饰和专属任务当前是第一轮主题化读链：它们会按村社 `theme + level` 生成可见内容和解锁状态，先承担“展示 / 预热 / 规划提示”，还不把更深的分配、投票和活动流程提前耦合进来。
- 在线 smoke 已补进 `L74` 回归：当前会实际验证公共仓入仓写路、服务端存档扣减、等级 / 福利回读和公共仓日志回读，村社权益第一轮已经进入可重复回归。
- 下一步进入 `L75`：围绕创建、加入、投票、公共建设和村社权益继承做 `L7` 回归验收，再决定是否继续深化专属权益的可操作流程。
