# 0518 联机执行锚点

## 1. 联机定义

`联机` 在桃源乡里不是“多人模式”按钮，也不是把个人存档直接改造成共享大房间。

当前统一定义为：

- 单人经营主循环仍然留在个人存档里。
- 跨玩家互动进入服务端权威的在线社会层。
- 奖励、来访、协作、节会与治理都通过联机事件和结算凭证回流个人成长。

这条定义是后续所有联机任务的硬边界。

## 2. 五大统一命名

后续联机 UI、文档、runtime、store 和事件命名，统一收口到这五条主线：

- `庄园`：公开展示、来访、留言、导览、收藏。
- `邻里`：名片、好友、关注、邻里关系、动态。
- `委托`：求助单、协作单、送礼、书信、慢交易。
- `节会`：活动日历、实时房间、节庆纪念、同场协作。
- `村社`：组织、提案、投票、公共建设、共治。

## 3. 五层宇宙到执行目录

`0518联机plan.md` 的五层联机宇宙，现统一拆成 `docs/online/` 下的执行入口：

- `00-base-audit.md`：当前在线底座盘点，先确认能复用什么。
- `06-naming.md`：在线层命名、runtime 切分、store 规范与事件命名。
- `07-state-and-settlement.md`：状态边界、幂等规则、冲突优先级与补偿策略。
- `08-room-state-machine.md`：实时房间统一状态机、成员状态与结算关闭流程。
- `01-manor.md`：庄园线，承接“可见联机”与来访痕迹。
- `02-neighbor.md`：邻里线，承接名片、好友、邻里与关系标签。
- `03-orders.md`：委托线，承接求助、送礼、书信和慢交易。
- `04-festival.md`：节会线，承接房间、节庆玩法与实时结算。
- `05-society.md`：村社线，承接组织、投票、公共建设与共治。

## 4. 入口顺序

后续推进顺序固定为：

1. 先做能留下痕迹的 `庄园 / 邻里 / 委托`。
2. 再做需要统一房间状态机的 `节会`。
3. 最后把长期组织和世界治理收进 `村社`。

## 5. 当前阶段约束

- 不把 `useGameStore` 这类单人运行态直接改成多人共享状态。
- 所有跨玩家奖励都必须走幂等、审计、补偿、断线恢复设计。
- 任何联机入口都要自然长进现有游戏世界，而不是额外挂一个孤立菜单。
- 命名、runtime 和事件统一遵循 `06-naming.md`，不再临时起名。
- 状态边界和发奖规则统一遵循 `07-state-and-settlement.md`，不允许后续任务重发明。
- 实时房间统一遵循 `08-room-state-machine.md`，不再为每个活动重做生命周期。
