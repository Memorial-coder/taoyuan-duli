# 节会线

- 主目标：在不破坏日常经营主循环的前提下，把同场联机收进节庆与短时活动房间。
- 首批范围：统一房间状态机、Ready、倒计时、断线重连、结算凭证。
- 首批目标文件：`server/src/taoyuanActivityRoomRuntime.js`、`taoyuan-main/src/views/game/FestivalView.vue`、`taoyuan-main/src/stores/useFestivalRoomStore.ts`、`taoyuan-main/src/utils/festivalRoomApi.ts`。
- 边界：房间负责互动过程，奖励通过结算凭证安全回写个人存档。

## 当前进度

- `L60` 第一轮已完成：节会房间现在已经支持创建、邀请、加入、准备确认、倒计时、开场演出、断线重连、结算凭证与关闭流程。
- `L61` 第一轮已完成：元日守岁、上元灯会、端午赛舟、七夕同游、中秋赏月、腊八共煮六个节会房型已经接入同一套房间模板。
- `L62` 第一轮已完成：公共进度、小队协作、抢答、拼装、采集、表演、合照七类玩法模板已经挂入房间运行态，并能通过统一动作接口推进共享进度、成员贡献和团队分数。
- `L63` 第一轮已完成：参与奖励、协作奖励、排名奖励、纪念奖励、限定装饰奖励和节气称号奖励已经接入逐成员结算凭证，并在房间关闭阶段真实写回个人存档与公开档案。
- 当前边界仍保持不变：`L63` 虽然已经落下真钱、纪念券、装饰和称号，但依旧通过 `settle -> pending_persist receipt -> close -> persisted` 这条链执行，没有把奖励提前塞回房间过程态。
- `L64` 第一轮已完成：节会奖励在真实写回个人存档时，会同步生成节会纪念册条目，记录房型、玩法模板、奖励摘要、同场成员、同场好友和合照文案；节会页右侧也已补出“最近纪念册”回看区。
- `L65` 第一轮回归已通过：在线 smoke 现在会实际验证房间稳定进出、断线重连、玩法推进、结算落档、纪念册回读，以及关闭后二次 `close` 被拒绝，确保不会重复发奖。
- 下一步进入 `L6` 深化：在现有纪念册和回归底座之上继续补更细的节会素材、专属演出、题库内容和更丰富的节会奖励池，而不是回头重做房间状态机。
