# 0510 计划：系统化借鉴《星露谷物语》玩法的长期扩展方案

## 0. 文档定位

这份文档的目标，不是“再多加一个种地子系统”，而是把最值得借鉴的《星露谷物语》长期钩子，系统性移植到《桃源乡独立版》当前已经很厚、但仍偏任务直给的中后期生活层。

本轮重点不再是泛泛头脑风暴，而是把真正高价值的长期循环正式变成：

- 可执行主线
- 可落地模块
- 可拆分任务
- 可回写的验收标准

默认约束：

- 不考虑工作量上限。
- 默认按依赖顺序推进。
- 默认优先复用现有 `data / stores / views / components` 结构。
- 每完成一个大阶段，都要更新 `0510todo.md` 与两个 `CHANGELOG`。

## 1. 这轮真正要借鉴的，不是“多做种地”

再筛一轮后，最值得继续借鉴的不是“再多一个种地子系统”，而是下面这些《星露谷物语》式长期钩子。

### 1.1 优先级最高的 12 个玩法

1. 书商 / 书籍系统  
每季低频来访，卖永久小增益、技能书、见闻书、线索书。它提供的是“低频但必须关注”的日历事件价值。

2. 奖券 + 兑奖机  
任务、特殊订单、节日优胜、连续成就给奖券，再去固定地点抽取预排奖池。它非常适合承接当前已经很强的任务板。

3. 神秘箱 / 神秘包裹  
通过钓鱼、挖矿、砍树、怪物、节日、特殊商人等多入口掉落，再去固定 NPC 或设施统一开箱。适合做成“桃源密匣”。

4. 小型钓鱼节 / 季节竞赛  
不是大型节日，而是 1-3 天的短活动窗口，强迫玩家轻度调整这几天的节奏。

5. 节日第二年变化  
同一个节日第二年起，摊位、对话、奖池、参赛物、地图布置发生变化。

6. 宠物深化  
多宠物、宠物送礼、宠物碗/窝升级、不同季节和关系阶段的反馈。

7. 家居扩建 / 房屋追加房间  
扩建不只是装饰，而要解锁新功能位、新展示位、新关系事件位。

8. 终局精通洞窟 / 大师系统  
当前已经有技能和 Perk，但还缺“五系练起来之后”的汇流终点。

9. 随机祝福 / 每日神像加成  
每天给一条轻偏向，推动玩家今天更倾向钓鱼、挖矿、节庆交付或采集。

10. 战斗饰物 / 小挂件槽位  
不做大装备树，只做一层轻 Build，让战斗、探索、关系、稀有来访预感多一层搭配。

11. 新住户 / 新邻居解锁线  
通过村庄建设、特殊订单、长期线索把新角色、新商店、新对话群引进村子。

12. 位置感更强的世界地图  
不是另起一页孤岛地图，而是带季节版、事件版、访客版、修复版的动态村图与行旅图联动。

### 1.2 很适合做成《桃源乡》变体的 10 个点

- 书商 -> 游学书肆 / 行脚书生 / 山门藏书摊
- 奖券兑奖机 -> 乡约牌 / 祠堂赏格 / 村衙赏契
- 神秘箱 -> 密匣 / 山泽遗箱 / 灵物封匣
- 小型钓鱼节 -> 鱼汛周 / 河鲜赛 / 夜钓会
- 第二年节日变化 -> 结合村庄建设等级 / NPC 关系 / 当季主题周
- 新住户线 -> 商队定居 / 学者驻村 / 山灵化人
- 每日祝福 -> 祠堂签 / 山神兆 / 节气气运
- 宠物送礼 -> 灵宠衔物 / 田犬报喜 / 猫叼线索
- 战斗饰物 -> 护符 / 玉佩 / 灵器碎片
- 动态地图 -> 与现有行旅图 / 村庄建设 / 节庆联动，而不是新开孤岛页

### 1.3 不建议优先重复抄的内容

- 旅行商人：已经有了，更应该强化预告、期待感、稀有货偏置。
- 祠堂收集包 / 社区目标：已经有了，更应该强化“修完世界真的变了”。
- 秘密笔记：已经有了，更应该改成“线索 -> 验证 -> 记录”，不是再加几十条静态文本。
- 脱水机 / 熏鱼：已有类似加工线，不值得再当主卖点。
- 技能分支：当前已经不弱，更该做“终局精通汇流”。

### 1.4 可作为扩展补强的 8 个原创方向

- 礼物线索日志
- 节日前备货清单
- 村民会买你卖给商店的东西
- 季节性街景变化
- 重复夺冠的隐藏奖励
- 传闻类每日轻任务
- 收集类大奖章页
- 可期待但不强迫的来访节奏

## 2. 当前项目现状映射

当前项目最该补的，不是系统数量，而是这些结构层缺口：

1. 社区修复后的真实世界变化
2. 日历驱动的准备感
3. NPC 喜好与关系的渐进发现
4. 秘密纸条的链式验证价值
5. 更角色化的特殊订单
6. 年度 / 双年度节日变化
7. 多技能汇流精通洞窟 / 大师系统
8. 环境事件 / 季节异象 / 天气窗口的真实计划价值
9. 奖券兑奖与阶段奖池循环
10. 神秘箱 / 密匣开箱循环
11. 宠物深化与家居扩建
12. 每日祝福 / 护符 / 轻构筑位
13. 新邻居与动态地图反馈

## 3. 玩法扩展总图

建议把整个 0510 计划拆成 8 条主线。

### WS01 社区修复 2.0

目标：

- 把祠堂收集包与村庄建设从“领奖励”升级成“改变世界”。

核心借鉴：

- Community Center
- Bus / Minecart / Bridge / Greenhouse

### WS02 日历、稀有访客与短活动 2.0

目标：

- 让日历真正驱动准备。
- 引入书商、小型节庆、低频高期待来访。

核心借鉴：

- Bookseller
- Traveling Cart
- Night Market
- 小型活动窗口

### WS03 NPC 关系与礼物发现 2.0

目标：

- 让礼物与关系从查表系统升级为观察与验证系统。

核心借鉴：

- Gift discovery
- Dialogue clues
- Birthday prep loop

### WS04 秘密纸条、线索链与发现记录 2.0

目标：

- 把秘密纸条升级成“线索 -> 验证 -> 记录 -> 回报”的闭环。

核心借鉴：

- Secret Notes
- Hidden clue chains

### WS05 特殊订单、主题周与公告板 2.0

目标：

- 把特殊订单做成人物化、世界化、阶段化的长期内容。

核心借鉴：

- Special Orders Board
- 高阶段挑战订单

### WS06 终局精通 2.0

目标：

- 做一个“五系都练起来之后”的汇流终点。

核心借鉴：

- Mastery Cave
- Post-max progression

### WS07 活世界反馈 2.0

目标：

- 让季节、天气、事件、节日、关系、建设状态共同改变玩家决策。

核心借鉴：

- Event windows
- Year-over-year variation
- Environment planning value

### WS08 生活设施、奖励循环与家园成长 2.0

目标：

- 把奖券兑奖、神秘箱、宠物、家居扩建、每日祝福、护符饰物、新邻居、动态地图这些很“星露谷”、但又不该散着做的内容收成一条完整主线。

核心借鉴：

- Prize Tickets / Prize Machine
- Mystery Box
- Pet deepening
- Home renovations
- Daily blessings
- Trinkets
- New neighbors
- Dynamic world map

## 4. 各主线详细设计

## 4.1 WS01 社区修复 2.0

重点做法：

- 社区收集包与村庄建设强绑定，不再各自独立。
- 每个修复成果至少带来一项真实日常便利。
- 每个修复成果至少产生四层反馈：
  - UI 变化
  - 页面入口变化
  - NPC 对话变化
  - 地图或路线变化

优先模块：

- 交通 / 捷径类设施
- 生产便利类设施
- 稀有访客 / 新商铺类设施

## 4.2 WS02 日历、稀有访客与短活动 2.0

### 核心模块

#### A. 书商 / 书籍系统

- 每季固定低频来访
- 日历提前标记
- 桃源乡命名建议：
  - 游学书肆
  - 行脚书生
  - 山门藏书摊
- 书籍类型至少包括：
  - 永久小增益书
  - 技能书
  - 见闻书
  - 线索书

#### B. 提前准备提醒

- 生日前 2 天提醒
- 节日前 2-3 天提醒
- 稀有访客日前提醒
- 天气窗口提醒
- 节日前备货清单自动生成

提醒内容至少覆盖：

- 这几天更值得准备的物资
- 对应角色这几天更值得推进的关系
- 今天错过就要等很久的内容
- 这次节日前值得优先囤哪些货

#### C. 小型季节竞赛与短活动窗口

- 1-3 天鱼汛赛 / 夜钓会
- 限时采集周
- 夜市式短来访
- 节前试吃 / 节前试钓 / 节前小摊预热
- 桃源乡命名建议：
  - 鱼汛周
  - 河鲜赛
  - 夜钓会

设计原则：

- 不抢大型节日风头
- 不做高压打卡
- 但足够让玩家临时改一两天节奏
- 让玩家愿意“等那天”，但不形成高压签到义务

#### D. 节日第二年变化

- 第二年起摊位变化
- 第二年起对话变化
- 第二年起奖池变化
- 第二年起参赛内容或地图布置变化
- 变化条件优先接：
  - 村庄建设等级
  - NPC 关系状态
  - 当季主题周
- 连续夺冠时补隐藏奖励或特殊事件，形成长期节日追求

## 4.3 WS03 NPC 关系与礼物发现 2.0

核心模块：

- 礼物认知层级：未知 / 模糊线索 / 明确偏好 / 已验证最爱最厌
- 礼物线索日志
- 多来源线索池：
  - 日常对话
  - 节日对话
  - 居所描述
  - 秘密纸条
  - 商店文案
  - 其他 NPC 转述
- 核心体验是“靠对话、纸条、节日观察一点点解锁”，而不是直接显示完整喜好答案

增强点：

- 书商来访时高学识 NPC 有额外评论
- 常送某类礼物后，对话会承认这种关系发展

## 4.4 WS04 秘密纸条、线索链与发现记录 2.0

核心模块：

- 纸条改成线索链，不再直接给奖励
- 统一写回发现记录
- 分类至少包括：
  - 礼物线索
  - 藏宝线索
  - 地点谜题
  - 世界传闻
  - 人物秘闻

验证条件优先接：

- 时间
- 地点
- 天气
- 节日
- 关系状态

## 4.5 WS05 特殊订单、主题周与公告板 2.0

核心模块：

#### A. 角色化特殊订单

- 明确发布人
- 明确动机
- 明确阶段
- 明确长期回响

#### A1. 传闻类每日轻任务

- 比普通订单更短
- 更口语化
- 更像村里流言或临时消息
- 适合承接稀有访客、节前备货、天气窗口、街市波动

#### B. 奖券与阶段奖池循环

- 普通任务、特殊订单、节日优胜、连续成就都能给奖券
- 奖券去固定地点兑换，不直接变钱
- 桃源乡命名建议：
  - 乡约牌
  - 祠堂赏格
  - 村衙赏契
- 奖池分阶段推进，不完全黑箱随机
- 奖池至少覆盖：
  - 功能种子
  - 稀有家具
  - 神秘箱
  - 关系礼物
  - 家居扩建材料

#### C. 订单历史与完成勾记

- 已做过的订单要有记号
- 能回看完成记录
- 重复订单要有变体或更高阶版本

#### D. 高阶段特殊订单

- 高地连续远征
- 多系统连锁目标
- 大量高质量供货限定委托

## 4.6 WS06 终局精通 2.0

核心模块：

#### A. 五系主精通

- 农耕精通
- 钓鱼精通
- 采集精通
- 挖矿精通
- 战斗精通

#### B. 混合精通

- 行旅大师：采集 + 钓鱼 + 区域探索联动
- 考据大师：博物馆 + 纸条 + 隐藏线联动
- 生活大师：宠物 + 家居 + 祝福 + 奖券联动

#### C. 精通解锁的功能性奖励

- 每日祝福神像
- 护符 / 饰物槽位
- 高级工具或高级加工台
- 稀有资源转化配方
- 特殊地图标记能力

要求：

- 精通奖励不能只给抽象数值
- 必须能被玩家每天感知

## 4.7 WS07 活世界反馈 2.0

核心模块：

#### A. 年度 / 双年度节日变化

- 第二年开始摊位变化
- 节日文案与评论变化
- 奖品变化
- 参赛物变化
- 地图装点变化

#### B. 环境事件 / 季节异象 / 天气窗口

- 特定天气更适合钓鱼
- 特定天气更适合挖矿
- 特定季节河岸 / 山路 / 村口出现事件窗口
- 这些窗口必须有真实计划价值，而不是纯文本氛围

#### C. 街景、地图与商业回响

- 季节性街景变化
- 社区修复后出现新招牌、新摊位、新通路
- 玩家长期卖某类商品后，NPC 或商店会提到这些变化
- 村民会对你最近卖给商店的东西产生反馈，形成“最近村里什么在流行”的生活感
- 动态地图显示：
  - 当前季节变体
  - 来访气泡
  - 活动标记
  - 修复后设施位置

## 4.8 WS08 生活设施、奖励循环与家园成长 2.0

### 4.8.1 当前问题

- 当前项目已经很像一款“系统丰富的田园 RPG”，但还缺一批让玩家觉得“今天顺手看一眼也值”的生活边角循环。
- 这些内容单独看都不够大，容易一直被排到后面；但它们合在一起，恰好是《星露谷》后期最耐玩的生活层。

### 4.8.2 借鉴目标

把下面这些内容收进一个统一主线里：

- 奖券兑奖
- 神秘箱 / 密匣
- 宠物深化
- 家居扩建
- 每日祝福
- 护符 / 饰物
- 新邻居
- 动态地图

核心目标不是“加更多页”，而是增加：

- 低频期待
- 重复游玩的轻反馈
- 家园成长的长期满足
- 后期不靠主线也有很多小动力

### 4.8.3 具体可落地模块

#### A. 奖券与兑奖机

- 任务、特殊订单、节日优胜、连胜成就给奖券
- 固定地点提供兑奖机或赏契台
- 奖励池按阶段推进，不完全随机
- 奖池包含：
  - 功能性种子
  - 稀有家具
  - 家居扩建材料
  - 好礼道具
  - 神秘箱
  - 长线收藏物

#### B. 神秘箱 / 密匣循环

- 多入口掉落：钓鱼、挖矿、采集、怪物、节日、书商、奖券机
- 固定 NPC 或固定设施统一开启
- 桃源乡命名建议：
  - 密匣
  - 山泽遗箱
  - 灵物封匣
- 内容分层：
  - 普通补给
  - 季节种子
  - 稀有礼物
  - 见闻书
  - 家居图纸
  - 特殊线索物

#### C. 宠物深化与多宠物路线

- 宠物亲密度后期成长
- 宠物偶尔衔物 / 报信 / 带回轻奖励
- 桃源乡表现建议：
  - 灵宠衔物
  - 田犬报喜
  - 猫叼线索
- 多宠物收养条件
- 宠物窝、食盆、饰物位
- 节日和家居中的宠物互动反馈

#### D. 房屋扩建与功能性家居

- 新房间扩建
- 功能角落扩建
- 展示配偶、收藏、奖杯、纪念物的展示位
- 与书商、奖券、节日、社区修复联动的家具套组
- 部分家居提供轻功能，而不只是装饰

#### E. 每日祝福 / 占卜 / 神像加成

- 每日一条偏向性祝福
- 祝福来源可以是祠堂、神像、占签、山灵
- 桃源乡命名建议：
  - 祠堂签
  - 山神兆
  - 节气气运
- 祝福影响至少覆盖：
  - 钓鱼收益
  - 采集发现率
  - 挖矿特殊掉落
  - 节庆交付
  - NPC 互动收益

#### F. 护符 / 饰物 / 小构筑位

- 不做完整大装备树，而做轻量构筑位
- 来源可以是精通、奖券、节日、神秘箱、隐藏线
- 桃源乡命名建议：
  - 护符
  - 玉佩
  - 灵器碎片
- 护符影响至少覆盖：
  - 战斗补正
  - 远征轻增益
  - 关系互动加成
  - 稀有来访预感

#### G. 新邻居 / 新住户线

- 通过社区修复、特殊订单或长期线索引入新住户
- 桃源乡方向建议：
  - 商队定居
  - 学者驻村
  - 山灵化人
- 新住户不一定都是婚恋对象，也可以是：

8. WS07 活世界反馈 2.0

## 7. 约束与反模式

- 不允许大量新增同义系统页，造成玩法分裂。
- 不允许把探索线索直接全部显式化。
- 不允许做纯数值奖励而没有世界反馈。
- 不允许把奖券、密匣、宠物、家居、祝福做成互相不认识的零散小系统。

进一步约束：

- 书商不能只卖书，必须接技能 / 百科 / 关系 / 节日至少两项。
- 纸条不能只给奖励，必须接地点 / 人物 / 社区 / 天气至少两项。
- 奖券不能只给随机物，必须接任务 / 节日 / 家居 / 收藏至少两项。
- 宠物不能只做摆设，必须接家园 / 节日 / 轻奖励至少两项。
