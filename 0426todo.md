## 4. 推荐实施顺序（给本轮开发）

1. [x] **先做 T02：顶部全局信息层压缩（升级为三段制）**  
   原因：当前最强的拥挤感来自“全局信息层 + 页面说明层”共同挤占首屏，必须先把首屏秩序理顺。

2. [x] **再做 T03：悬浮按钮 + 地图菜单改单入口**  
   原因：右侧五连按钮是移动端持续视觉噪音源，且会和正文争抢注意力，属于高收益全局收口项。

3. [x] **然后做 T11：商店 / 任务页主操作前置**  
   原因：这两个页面是高频页面，最容易暴露“先读说明再操作”的问题，改完后体感提升最直接。

4. [x] **再做 T06：行旅图任务流重排**  
   原因：行旅图现在最像“桌面结构压缩版”，应在全局层和高频页收口后，再正式重排成流程页。

5. [ ] **最后做 T12 + T13：密度规范与主引导去重**  
   原因：这是对前面改动的统一收尾，确保后续不会重新长回“多入口、多话术、多说明层”状态。
